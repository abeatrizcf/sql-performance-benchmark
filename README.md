## Projeto 3 -  Comparação de Estratégias SQL para Otimização de Consultas  

### Problema: 
Dentro do Universo do SQL, existem diversas formas possíveis para realizar um mesmo problema. Busca-se avaliar se, de fato, métodos mais simples e iniciais são menos eficientes do que métodos avançados em todos os casos.
Solução:

Para compreender essa questão, foi desenvolvido uma tabela com Python contendo 200.000 registros de 2000 clientes e almeja-se analisar o tempo necessário para calcular a média das últimas 3 compras de cada cliente. Será avaliado um total de quatro estratégias: Subqueries relacionadas (com e sem o uso de index) e Windows Functions (com e sem o uso de index)

### Objetivos:
Comparar o tempo de execução de diferentes estratégias SQL para resolver um mesmo problema.
Analisar o impacto do uso de Subqueries Correlacionadas, Window Functions e Índices no desempenho das consultas.
Avaliar como cada abordagem influencia o plano de execução por meio do EXPLAIN QUERY PLAN.
Identificar a estratégia mais eficiente para o cenário proposto com base em métricas de desempenho.

### Habilidades técnicas demonstradas:
SQL básico
SQL Avançado
SQLite
Subqueries Correlacionadas
Window Functions
Índices
EXPLAIN QUERY PLAN
Otimização de Consultas
Performance em Banco de Dados
